# al-co — План разработки MVP

> **Вайб:** как в детстве собраться у друга с одной консолью, орать на телек и пихаться за геймпад. Только теперь с пивом.

---

**Входит в MVP:**

- Подключение 4–8 игроков с телефонов по QR-коду
- Колода контента: вопросы, задания, карточки внешних игр
- 2 встроенных мини-игры с управлением с телефона
- Динамическая смена контроллера на телефоне под конкретную игру
- Ноутбук как главный экран + кнопка "следующее"

**Не входит в MVP:**

- Счётчик выпитого
- Система очков
- Больше 2 встроенных мини-игр
- Настройки (сложность, категории)
- Красивый UI/дизайн
- Больше 30–40 вопросов и заданий

---

## Архитектура

```
[Node.js + Express + Socket.io]
         ↕
[Ноутбук — главный экран]    [Телефон каждого игрока]
 Phaser.js (мини-игры)        nipplejs / кнопки / tap
 HTML/CSS (UI карточек)       HTML/CSS (контроллер)
```

**Правило:** вся логика состояния — на сервере. Клиенты только отображают и отправляют ввод.

## Контент-стратегия

### Три типа карточек

\*\*1. Вопрос\*\*

```json
{
  "type": "question",
  "text": "Сколько?",
  "answers": \\\["228", "67", "1488", "0"],
  "penalty": "Не угадал — пьёшь"
}
```

Ответы зачитываются вслух, правильных может быть несколько. Появляется поле для ввода, далее идёт сравнение с ответом и результат - "правильно!", "не правильно...".

**2. Задание**

```json
{
  "type": "task",
  "text": "Говоришь только на английском до следующего задания",
  "penalty": "Сказал по-русски — пьёшь глоток"
}
```

Без программной проверки. Контроль — сами игроки. После выполнения, на главном экране нужно нажать "продолжить" когда все игроки будут готовы.

**3. Карточка внешней игры**

```json
{
  "type": "external",
  "game": "Gang Beasts",
  "image": "gang-beasts.jpg",
  "rules": "Каждый нокаут — пьёшь. Победитель раунда — раздаёт 2 глотка любому.",
  "note": "Запустить вручную. Играют все или по дуэльной системе."
}
```

Ноутбук показывает карточку. Ты запускаешь игру вручную. После игры/выполнения задачи, жмёшь "следующее".

**Список кандидатов для external:** Gang Beasts, Stick Fight, Mortal Combat, Overcooked

## Socket.io события

|Событие|Откуда|Куда|Описание|
|-|-|-|-|
|`create-room`|ноутбук|сервер|создать комнату, получить код|
|`join-room`|телефон|сервер|войти с именем и цветом|
|`player-joined`|сервер|ноутбук|обновить список игроков|
|`next-card`|ноутбук|сервер|показать следующий контент|
|`card-update`|сервер|все|тип и данные текущей карточки|
|`controller-change`|сервер|телефон|какой интерфейс показать|
|`controller-input`|телефон|сервер|ввод (джойстик/кнопка/тап)|
|`game-input`|сервер|ноутбук|транслировать ввод в игру|
|`mini-game-start`|сервер|все|запустить мини-игру|

Формат `controller-change`:

```json
{ "layout": "joystick" }
{ "layout": "buttons", "labels": \\\["УДАР", "БЛОК"] }
{ "layout": "tap", "count": 6, "colors": \\\["red","blue",...] }
{ "layout": "idle", "message": "Смотри на экран" }
```

## Две мини-игры для MVP

### Мини-игра 1 — Реакция (проще, делается первой)

Суть: на экране ноутбука появляется цветной круг с именем игрока. Этот игрок должен как можно быстрее тапнуть на телефоне. Кто медленнее всех — пьёт.

Контроллер: `{ "layout": "tap", "count": 1 }` — одна большая кнопка.

Почему первая: минимальная логика, чистый тест всей цепочки Socket.io → Phaser → контроллер.

### Мини-игра 2 — Танки

Суть: танчики из 90-х. Каждый игрок управляет своим танком. Попадание = соперник пьёт, пока пьёт — не может двигаться. У танка 3 жизни.

Контроллер: `{ "layout": "joystick" }` — nipplejs + кнопка выстрела.

Phaser.js сцена: Arcade physics, tilemap для стен, спрайты танков. Всё это есть в готовых примерах на phaser.io/examples.

## Этапы разработки

### Фаза 1 — Фундамент (1–2 недели)

* \[ ] Инициализация проекта: `npm init`, установка express + socket.io
* \[ ] Сервер: создание комнаты, написание кода, хранение списка игроков
* \[ ] Ноутбук: страница лобби, отображение кода комнаты и QR
* \[ ] Телефон: страница входа (имя + цвет), подключение к комнате
* \[ ] Трансляция события `player-joined` на ноутбук

**Результат:** 4 человека могут подключиться к одной комнате и видеть друг друга на экране ноутбука.

\---

### Фаза 2 — Движок контента (1–2 недели)

* \[ ] JSON-файлы с вопросами (20 штук), заданиями (20 штук), external (5 штук)
* \[ ] `contentLoader.js`: загрузить и перемешать колоду
* \[ ] `gameFlow.js`: выдать следующую карточку по запросу
* \[ ] Ноутбук: отображение карточки (вопрос / задание / external)
* \[ ] Кнопка "Следующее" на ноутбуке
* \[ ] Телефоны: layout `idle` пока идёт вопрос/задание

**Результат:** полноценная сессия вопросов и заданий. Уже можно играть вчетвером.

\---

### Фаза 3 — Первая мини-игра: Реакция (1 неделя)

* \[ ] Подключить Phaser.js на странице ноутбука
* \[ ] Сцена `ReactionScene`: таймер, случайный игрок, отображение круга
* \[ ] Сервер: отправить `controller-change` с `layout: "tap"` нужному игроку
* \[ ] Телефон: отобразить большую кнопку, отправить `controller-input` при тапе
* \[ ] Сервер: принять ввод, передать в игру, засчитать результат
* \[ ] По окончании: показать кто медленнее всех и правило выпивки
* \[ ] Добавить `"type": "minigame"` карточки в колоду

**Результат:** первая интерактивная мини-игра с телефоном работает.

\---

### Фаза 4 — Вторая мини-игра: Танки (2 недели)

* \[ ] Найти open-source Phaser3 tanks пример (phaser.io/examples или GitHub)
* \[ ] Адаптировать: убрать клавиатуру, добавить приём `game-input` через Socket.io
* \[ ] nipplejs на телефоне: джойстик + кнопка выстрела
* \[ ] Сервер: маршрутизировать ввод от каждого телефона к нужному танку
* \[ ] Правило выпивки: попал → соперник пьёт и замирает пока пьёт
* \[ ] Ограничить до 4 игроков (остальные смотрят)

**Результат:** MVP готов.

\---

### Фаза 5 — Шлифовка перед первым запуском (1 неделя)

* \[ ] Дизайн
