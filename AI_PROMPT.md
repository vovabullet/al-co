# al-co — Технический промт для AI-ассистента

Ты помогаешь разрабатывать **al-co** — локальную вечеринковую drinking game, которая запускается на ноутбуке, а игроки подключаются с телефонов через браузер по QR-коду в локальной сети. Никакого деплоя в интернет, никакого облака.

---

## Стек (не обсуждается, не предлагай замены)

**Бэкенд:** Node.js + Express + Socket.io  
**Главный экран (ноутбук):** Phaser.js (мини-игры) + ванильный HTML/CSS/JS (UI)  
**Контроллер (телефон):** ванильный HTML/CSS/JS + nipplejs (джойстик)  
**Сборщики:** никаких. Никакого Webpack, Vite, React, TypeScript. Чистые файлы.  
**CDN-библиотеки (подключаются тегом script):** Phaser 3, nipplejs, socket.io-client  

---

## Ключевые ограничения

- **Одна сессия = весь жизненный цикл приложения.** Запустили `npm start` → сыграли → выключили. Никаких комнат, никакого multi-room.
- **Состояние только на сервере.** Клиенты только рендерят и отправляют ввод.
- **Правила выпивки — просто текст на экране.** Никакого программного подсчёта глотков или очков.
- **Никакой авторизации, БД, аккаунтов.**

## Модель данных

### Состояние игры (`state.js`)

```js
const state = {
  phase: 'lobby',        // 'lobby' | 'playing' | 'minigame'
  players: [],           // [{ id, name, color, socketId }]
  deck: [],              // перемешанный массив карточек
  currentIndex: 0,       // индекс текущей карточки
  currentMiniGame: null  // null или название активной мини-игры

```

### Игрок

```js
{ id: 'uuid', name: 'Вася', color: '#e74c3c', socketId: '...' }
```

### Карточки контента

```js
// Вопрос
{ "type": "question", "text": "Сколько?", "answers": ["228", "67", "0"], "penalty": "Не угадал — пьёшь" }

// Задание
```

## Socket.io контракт (полный)

### Desktop → Server
| Событие | Payload | Описание |
|---|---|---|
| `start-game` | — | начать игру из лобби |
| `next-card` | — | перейти к следующей карточке |

### Phone → Server
| Событие | Payload | Описание |
|---|---|---|
| `join` | `{ name, color }` | подключиться к игре |
| `controller-input` | `{ type, data }` | ввод с контроллера |

### Server → Desktop
| Событие | Payload | Описание |
|---|---|---|
| `player-joined` | `player` | новый игрок подключился |
| `player-left` | `{ id }` | игрок отключился |
| `card-update` | `card` | показать новую карточку |
| `minigame-start` | `{ name, players }` | запустить мини-игру |
| `minigame-input` | `{ playerId, type, data }` | ввод от игрока в мини-игру |

### Server → Phone (конкретному сокету)
| Событие | Payload | Описание |
|---|---|---|
| `joined` | `{ player, players }` | подтверждение, текущий список |
| `controller-change` | `{ layout, ...options }` | сменить интерфейс контроллера |

### Форматы `controller-change`
```js
{ layout: 'idle', message: 'Смотри на экран' }
{ layout: 'tap', label: 'ЖМИ' }                        // одна большая кнопка
{ layout: 'joystick' }                                   // nipplejs + кнопка действия
{ layout: 'buttons', labels: ['УДАР', 'БЛОК'] }
```

## Мини-игры MVP

### Реакция
- Сервер выбирает случайного игрока, отправляет его телефону `controller-change { layout: 'tap' }`
- На десктопе Phaser показывает круг цвета игрока и запускает таймер
- Игрок тапает → сервер фиксирует время → повторяется для каждого
- Кто медленнее всех → показать результат + правило выпивки

### Танки
- Каждому игроку (макс 4) отправляется `controller-change { layout: 'joystick' }`
- Phaser сцена: вид сверху, стены, снаряды, Arcade physics
- `controller-input { type: 'move', data: { x, y } }` → двигать танк
- `controller-input { type: 'shoot' }` → выстрел
- Попадание → пострадавший замирает на 3 сек (выпивает), у танка 3 жизни
- Остальные игроки (5-8) смотрят с `{ layout: 'idle' }`

## Фазы разработки

### Фаза 1 — Фундамент ✅ делаем сейчас
1. `package.json`, установить `express socket.io qrcode`
2. `server/index.js`: Express раздаёт `/client`, Socket.io слушает `join` и `start-game`
3. `server/state.js`: объект состояния, функции `addPlayer`, `removePlayer`
4. `client/desktop/index.html`: лобби — список игроков + QR-код + кнопка "Начать"
5. `client/controller/index.html`: форма входа (имя + выбор цвета) → после `join` показывает "Ожидание..."
6. QR генерируется на локальный IP через пакет `qrcode`, отдаётся как base64 img

**Готово когда:** 4 телефона подключились, все видят друг друга на экране ноутбука.

---

### Фаза 2 — Движок контента
1. `content/questions.json` (20 штук), `tasks.json` (20 штук), `external-games.json` (5 штук)
2. `server/contentLoader.js`: читает JSON, перемешивает в одну колоду
3. `server/gameFlow.js`: `nextCard()` возвращает следующую карточку
4. После `start-game`: сервер переходит в `playing`, рассылает первую карточку
5. `client/desktop`: рендерит карточку в зависимости от `type`, кнопка "Следующее"
6. Телефоны получают `controller-change { layout: 'idle' }` пока идут вопросы/задания

**Готово когда:** можно провести полноценную сессию вопросов и заданий.

---

### Фаза 3 — Мини-игра: Реакция
1. Подключить Phaser 3 через CDN в `client/desktop/index.html`
2. `client/desktop/game.js`: создать Phaser.Game, зарегистрировать `ReactionScene`
3. Реализовать `ReactionScene` (таймер, визуал, ожидание ввода)
4. Сервер при карточке `{ type: 'minigame', name: 'reaction' }` → запускает игру, рассылает `controller-change`
5. Обработать `controller-input` → передать в сцену через `minigame-input`

**Готово когда:** реакция запускается из колоды, телефон реагирует, результат показывается.

---

### Фаза 4 — Мини-игра: Танки
1. Найти готовый пример Phaser3 top-down tanks (phaser.io/examples или GitHub)
2. Убрать клавиатурное управление, заменить на приём `minigame-input` через Socket.io
3. nipplejs на контроллере: джойстик + кнопка выстрела
4. Сервер маршрутизирует ввод каждого телефона к нужному танку
5. Логика заморозки при попадании

**Готово когда:** 2-4 человека играют в танки с телефонов.
